인가코드 발급 요청

v0 2025-10-24

기본 정보

API ID개별인증-001HTTP MethodGET
API 제공자정보제공자API 요청자마이데이터사업자
API 명 (URI)/oauth/2.0/authorize
설명정보주체(고객)가 마이데이터사업자 앱을 통해 개별인증수단(정보제공자가 제공)을 이용하여 인증 및 전송요구를 수행한 후 인가코드를 발급
- 정보제공자는 개별인증을 위한 인증화면 및 전송요구를 위한 자산선택 화면을 웹뷰 등으로 제공, 해당 화면을 통해 개별인증 및 전송요구 수행
- 인가코드 발급 후, 정보제공자는 redirect_uri (Callback URL)로 인가코드 등을 리다이렉트
기준시점현재 시점Content-Type (요청)-
Content-Type (응답)application/json

요청 메시지 명세

HTTP 항목명 항목설명 필수 타입(길이) 설명(비고)
Headerx-user-ci정보주체 식별값YB64(100)정보주체 식별을 위한 연계정보 (CI)
- 개별인증을 요청하는 정보주체의 CI값으로, 정보제공자는 보유한 CI값과 비교·검증해야 함
x-api-tran-id거래고유번호YAN(25)거래고유번호
Parameterorg_code기관코드YaN(10)정보제공자 기관코드
- 지원 API로부터 배포
response_type타입Ya(4)인가코드 반환을 의미
- 'code' 고정값
client_id클라이언트 IDYaN(50)종합포털에 마이데이터 서비스 등록 시 발급 받은 클라이언트 식별값
redirect_uriCallback URLYaNS(100)전송요구 후 응답이 전달(redirect)될 마이데이터 서비스 URI
- URL 인코딩 필요
app_scheme앱 URL 스킴YaNS(100)현재 실행중인 마이데이터서비스앱의 앱스킴
- 앱방식 개별인증 완료 후 마이데이터서비스 앱으로 전환 시 필요
- URL 인코딩 필요
state상태값YaN(40)CSRF 보안위협 대응을 위해 임의 설정하는 값

HTTP 항목명 항목설명 필수 타입(길이) 설명(비고)

Bodycode인가코드YaNS(128)발급한 인가코드(Authorization code)
- 유효시간 최대 10분 권고(RFC6749)
